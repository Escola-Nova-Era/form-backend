# 🚀 Desafio 05 — Testes com Jest

Neste desafio você irá testar regras de negócio da API para garantir confiabilidade e facilitar manutenção.

## 🎯 Objetivo do Desafio

Criar uma suíte de testes unitários para serviços críticos da aplicação.

## 🧠 Tecnologias que você irá praticar

- Jest
- Testes unitários
- Matchers
- Testes assíncronos
- Cobertura de testes

## 📦 Escopo funcional (MVP)

Cobrir pelo menos:

- criação de entidade válida;
- rejeição de payload inválido;
- atualização de dados;
- cenários de erro de negócio.

## 🛠 Requisitos técnicos

- Isolar regras em funções ou services testáveis.
- Criar testes com padrão `arrange, act, assert`.
- Testar fluxos síncronos e assíncronos.
- Gerar relatório de cobertura.

## ✅ Critérios de aceite

- Testes executam com `npm test`.
- Cenários principais cobertos.
- Falhas de regra são detectadas pelos testes.

## ⭐ Parte opcional

- Meta mínima de cobertura (ex.: 80% em services).
- Pipeline simples de teste em CI.

## 🏁 Resultado esperado

Ao concluir este desafio, você terá base prática de testes automatizados no backend com Jest.
