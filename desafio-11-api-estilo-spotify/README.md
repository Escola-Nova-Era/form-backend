# 🚀 Desafio 11 — API estilo Spotify

Neste desafio você irá desenvolver uma API com domínio mais rico, simulando catálogo e reprodução de música.

## 🎯 Objetivo do Desafio

Criar uma API com entidades de música, artista, álbum e playlist, com foco em modelagem e experiência de uso.

## 🧠 Tecnologias que você irá praticar

- Node.js/NestJS
- Modelagem relacional ou híbrida
- Regras de domínio
- Busca e filtros
- API REST avançada

## 📦 Escopo funcional (MVP)

Entidades mínimas:

- `artists`
- `albums`
- `tracks`
- `playlists`

Funcionalidades:

- CRUD de catálogo;
- adicionar/remover faixa em playlist;
- listar playlist com duração total;
- busca por nome de faixa/artista.

## 🛠 Requisitos técnicos

- Definir relacionamentos claros entre entidades.
- Implementar paginação em listagens principais.
- Tratar conflitos (faixa duplicada na playlist, por exemplo).
- Retornar payload consistente entre endpoints.

## ✅ Critérios de aceite

- API cobre operações de catálogo e playlists.
- Regras de domínio respeitadas.
- Consultas retornam dados estruturados e úteis.

## ⭐ Parte opcional

- Ranking de faixas mais adicionadas.
- Endpoint de recomendação simples por gênero.

## 🏁 Resultado esperado

Ao concluir este desafio, você terá experiência com APIs de domínio complexo e regras encadeadas.
