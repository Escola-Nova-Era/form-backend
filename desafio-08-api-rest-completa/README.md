# 🚀 Desafio 08 — API REST completa

Neste desafio você irá consolidar tudo que aprendeu até aqui em uma API mais robusta e pronta para uso real.

## 🎯 Objetivo do Desafio

Implementar uma API de gestão de cursos com autenticação básica, paginação, filtros e documentação mínima.

## 🧠 Tecnologias que você irá praticar

- Node.js ou NestJS
- REST avançado
- Validação de dados
- Paginação e filtros
- Tratamento padronizado de erros

## 📦 Escopo funcional (MVP)

Entidades sugeridas:

- `users`
- `courses`
- `enrollments`

Funcionalidades:

- CRUD de cursos;
- matrícula de usuários;
- listagem paginada;
- filtro por categoria e nível.

## 🛠 Requisitos técnicos

- Definir contratos claros de request/response.
- Aplicar middleware/guard para rotas protegidas.
- Criar padrão de erro (`code`, `message`, `details`).
- Documentar endpoints no README.

## ✅ Critérios de aceite

- API funcional de ponta a ponta.
- Rotas protegidas respeitando permissões básicas.
- Paginação e filtros funcionando.

## ⭐ Parte opcional

- Swagger/OpenAPI.
- Versionamento de API.

## 🏁 Resultado esperado

Ao concluir este desafio, você terá uma API REST profissional com recursos essenciais para produção.
