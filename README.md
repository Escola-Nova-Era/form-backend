# 🚀 Formação Backend JS

Bem-vindo à **Formação Backend JS** — uma jornada prática para transformar você em um desenvolvedor backend pronto para o mercado.

Aqui você não vai apenas assistir aulas.
Você vai **construir sistemas reais**, organizados em desafios progressivos.

---

## 🎯 Objetivo da Formação

Ao longo da formação, você vai desenvolver as competências para:

* Construir APIs backend com JavaScript e Node.js
* Modelar e integrar bancos de dados SQL e NoSQL
* Aplicar boas práticas de arquitetura e organização de código
* Implementar autenticação e regras de segurança
* Criar ambientes com Docker para desenvolvimento e deploy
* Garantir qualidade com testes automatizados
* Entregar projetos backend completos, prontos para portfólio e mercado

---

## 🧰 Tecnologias da Formação (usadas nos desafios)

🟢 **Banco de dados SQL (MySQL)**

* MySQL
* SQL (DDL e DML)
* `CREATE`, `INSERT`, `SELECT`, `UPDATE`, `DELETE`
* Tipos de dados e alteração de colunas
* Chave primária (`PRIMARY KEY`)
* Chave estrangeira (`FOREIGN KEY`)
* Relacionamentos (1:1, 1:N, N:N e autorreferência)
* Regras `ON DELETE` e `ON UPDATE` (`CASCADE`, `RESTRICT`, `SET NULL`)
* Junções (`INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN`, `CROSS JOIN`)
* Funções SQL e formatação de datas

🟩 **Banco de dados NoSQL (MongoDB)**

* MongoDB
* Collections e documentos
* `find`, `update`, `delete`
* Operadores de comparação e lógicos
* Operadores de array
* Modelagem de relacionamentos (embutido e referenciado)
* Aggregation Pipeline (consultas agregadas)

🌐 **Backend e APIs**

* JavaScript (Node.js)
* TypeScript
* NestJS
* Arquitetura modular (`Module`, `Controller`, `Service`)
* DTO (Data Transfer Object)
* Métodos HTTP (`GET`, `POST`, `PUT`, `PATCH`, `DELETE`)
* Parâmetros de rota (`@Param`) e query (`@Query`)
* Body em `application/json`, `x-www-form-urlencoded` e `multipart/form-data`
* Códigos de status HTTP e tratamento de erros
* Validação de dados com Zod

🗃 **ORM e persistência**

* Prisma ORM
* Mapeamento de modelos e tabelas
* Relacionamentos no banco de dados
* Migrações
* Seeds
* Cliente de banco com padrão singleton

🐳 **Containers e ambiente**

* Docker
* Dockerfile
* Imagens e containers
* Volumes e bind mounts
* Networks
* Variáveis de ambiente
* Docker Compose
* Orquestração de múltiplos serviços

🔐 **Autenticação e segurança**

* Autenticação com credenciais
* Fluxo de login e logout
* NextAuth (Auth.js)
* Validação de entrada e retorno de erros padronizado
* 2FA (autenticação em dois fatores)

🧪 **Testes e qualidade**

* Jest
* Testes unitários
* Matchers
* Testes assíncronos com Promises
* Mocks e spies
* Cobertura de testes (coverage)

🏗 **Arquitetura e práticas profissionais**

* API REST
* Organização por camadas
* Separação de responsabilidades
* Padronização com ESLint e Prettier
* Versionamento com Git
* Gestão ágil com Scrum
* Fluxo de trabalho com Jira
